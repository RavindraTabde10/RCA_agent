# RCA Infotainment Module

Automated Root Cause Analysis for Automotive Infotainment Systems.

## Features

- **DLT Log Analysis** - Parse and analyze Diagnostic Log and Trace files
- **Historical Matching** - Find similar past defects for quick resolution
- **Duplicate Detection** - Automatically identify potential duplicate tickets
- **LLM-Powered Analysis** - AI-driven root cause identification
- **Source Code Mapping** - Map errors to affected source files
- **JIRA Integration** - Auto-comment, upload reports, link duplicates
- **Git Integration** - Pull source code from repository for analysis
- **Report Generation** - Markdown and HTML reports

## Quick Start

```bash
# Install dependencies
pip install -r requirements_rca.txt

# Check service status
python -m src.rca_infotainment.rca_cli status

# List defects
python -m src.rca_infotainment.rca_cli list

# Analyze a defect
python -m src.rca_infotainment.rca_cli analyze SAM1-2001

# Analyze and upload to JIRA
python -m src.rca_infotainment.rca_cli analyze SAM1-2001 --jira
```

## Configuration

### 1. Environment Variables

Copy `.env.example` to `.env` and configure your credentials:

```bash
cp .env.example .env
```

### 2. Configure Services

Edit `config/config.yaml` to enable services:

#### LLM (Required for AI analysis)

```yaml
llm:
  provider: "azure_openai"  # or "openai" or "custom"
```

Set environment variables:
- **Azure OpenAI**: `AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_API_KEY`, `AZURE_OPENAI_DEPLOYMENT_NAME`
- **OpenAI**: `OPENAI_API_KEY`
- **Custom**: `LLM_ENDPOINT_URL`, `LLM_API_KEY`

#### JIRA (Required for ticket integration)

```yaml
integrations:
  jira:
    enabled: true
```

Set environment variables:
- `JIRA_URL` - Your JIRA instance URL
- `JIRA_EMAIL` - Your Atlassian email
- `JIRA_API_TOKEN` - API token from Atlassian account
- `JIRA_PROJECT_KEY` - Project key (e.g., SAM1)

#### Git (Required for source code access)

```yaml
integrations:
  git:
    enabled: true
```

Set environment variables:
- `GIT_REPO_URL` - Repository URL (HTTPS or SSH)
- `GIT_BRANCH` - Branch name (default: main)
- `GIT_TOKEN` - Personal access token for private repos
- `GIT_LOCAL_PATH` - Local path to clone repo

## Output

The analysis generates:

1. **Markdown Report** (`output/reports/{defect_id}_rca.md`)
2. **HTML Report** (`output/reports/{defect_id}_rca.html`)
3. **JIRA Comment** (when `--jira` flag is used)
4. **JIRA Attachments** (MD + HTML reports)
5. **Duplicate Link** (if duplicate detected)

## JIRA Comment Format

```
h2. 🔍 AI Root Cause Analysis

h3. Root Cause
{Analysis result from LLM}

*Confidence:* 85%

h3. Fix Recommendation
{Specific code changes recommended}

h3. Evidence
{DLT log patterns and errors}

h3. Related Defects
* SAM1-123 - 78% similar
* SAM1-456 - 72% similar

----
_Generated by RCA Agent_
```

## Duplicate Detection

When a defect is **≥90% similar** to a historical defect:
- Marked as potential duplicate
- Original ticket linked
- Warning banner in report
- JIRA comment with recommendation

## Data Files

Place your data in:
- `data/defects/jira_api_defects.json` - Current defects to analyze
- `data/defects/historical_defects.json` - Historical defects for matching
- `data/dlt_logs/` - DLT log files
- `data/domain_knowledge/` - Domain knowledge markdown files

## CLI Commands

| Command | Description |
|---------|-------------|
| `list` | List all current defects |
| `stats` | Show defect statistics |
| `search "query"` | Search historical defects |
| `analyze <id>` | Analyze a defect |
| `analyze <id> --jira` | Analyze and upload to JIRA |
| `check_duplicate <id>` | Check if defect is duplicate |
| `view_report <id>` | View generated report |
| `status` | Check service status |

## Development

### Module Structure

```
src/rca_infotainment/
├── __init__.py           # Module exports
├── rca_cli.py            # CLI entry point
├── rca_engine.py         # Main orchestrator
├── dlt_analyzer.py       # DLT log parsing
├── source_mapper.py      # Error→code mapping
├── historical_matcher.py # Duplicate detection
├── report_generator.py   # MD/HTML reports
├── jira_service.py       # JIRA integration (placeholder)
├── llm_service.py        # LLM integration (placeholder)
└── git_service.py        # Git integration (placeholder)
```

### Adding New Services

1. Create service file in `src/rca_infotainment/`
2. Add `set_*_client()` method to `RCAEngine`
3. Initialize in `rca_cli.py` `get_engine()`
4. Update `status` command to show service status
5. Add config section to `config/config.yaml`
